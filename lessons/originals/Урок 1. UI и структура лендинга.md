Сначала немного теории - что из себя представляет лендинг страница? Это простой одностраничный сайт, размещаемый в глобальной сети, который служит главной цели - получать контакты потенциальных клиентов, чтобы дальше появилась возможность с ними работать.

Из чего состоит лендинг:
- Hero - кому и зачем этот продукт
- Proof - почему нам можно доверять
- Benefits - какую конкретно пользу получит пользователь
- FAQ - снятие возможных сомнений и возражений
- CTA - призыв к действию, ради которого и создавался лендинг

Теперь немного подробнее.

<пример Hero>
### Hero
Это первый экран, который видит пользователь. 
А значит здесь мы размещаем:
- основной заголовок, чтобы сразу обозначить общий контекст
- краткое пояснение по продукту - объяснить для кого этот продукт и зачем, чтобы заинтересовать пользователя и побудить к дальнейшему скроллингу страницы
- кнопка или ссылка, которая сразу ведет к CTA-блоку - чтобы пользователь имел возможность незамедлительно воспользоваться предложением

<пример Proof>
### Proof
Здесь мы должны убедить пользователя нам доверять. 
Поэтому тут размещаем:
- какие то конкретные факты и цифры
- примеры реальных кейсов
- клиентов, которые уже воспользовались продуктов

<пример Benefits>
### Benefits
Эта секция самая важная, так как в ней нужно донести до пользователя - что конкретно получит он, если воспользуется предложением, какие для него выгоды? 
Поэтому обычно это:
- 3–6 конкретных выгод
- короткое пояснение к каждой
- правильная подача - меньше текста и больше визуала

<пример FAQ>
### FAQ
Если пользователь дошёл до FAQ - он уже заинтересован, но всё ещё сомневается. Поэтому мы должны проговорить возможные страхи пользователя и снизить барьер к действию.
Какие можно включить вопросы:
- “Нужен ли опыт?”
- “Сколько времени это занимает?”
- “Есть ли скрытые условия?”
- “Что если мне не подойдёт?”

<пример CTA>
### CTA
Финальная секция, где пользователь принимает решение на совершение целевого действия.
Поэтому здесь:
- короткий заголовок
- повтор ценности (1–2 предложения)
- форма или кнопка
- минимально необходимое количество полей

А теперь, получив необходимые знания, давайте же попробуем самостоятельно создать свой лендинг!

Для этого нам потребуется:
- установить на нашем компьютере IDE Visual Studio Code <ссылка на отдельный урок>
- настроить нужные плагины для взаимодействия с выбранной LLM
- составить запросы на создание лендинга и передать их в LLM
- после чего мы сможем запустить в браузере получившийся результат

Для примера, создадим лендинг предварительной регистрации пользователей на будущий курс Яндекс.Практикум «Вайбкодинг».

Открываем окно запроса AI агента GPT-5.3-Codex и отправляем ему запрос:
```
Ты профессиональный интернет-маркетолог с навыками frontend-разработчика.  

Необходимо разработать лендинг предварительной регистрации пользователей на обучающий курс для маркетологов, проджект и продакт менеджеров под названием "Вайбкодинг".

Требования:
- лендинг должен состоять из блоков: Hero, Proof, Benefits, FAQ, CTA
- Hero содержит заголовок и призыв к действию в блоке CTA
- Proof содержит рандомные продающие показатели и кейсы реальных покупателей курса. Оставь плейсхолдеры под фото клиентов
- Benefits - придумай 3 подходящих под контекст выгод прохождения курса для потенциального студента с кратким пояснением по каждому
- FAQ - придумай 3 наиболее вероятных вопроса покупателя курса и подходящие ответы на них
- CTA должен содеражать форму отправки лида с полями: имя (строка), телефон (строка), чекбокс согласия обработки персональных данных и кнопку отправки формы с подходящим текстом
  
Дополнительные инструкции:
- каждый блок имеет якорь для навигации
- для написания лендинга использовать только: html, css
- результат работы разместить в файле index.html
- верстка страницы адаптивная
  
Работу выполняй поэтапно:
- разметь основную структуру страницы
- последовательно заполняй блоки
- соблюдай логическую связность между блоками - каждый последующий блок это борьба с потенциальными возражениями клиента и усилия на перевод клиента на CTA форму
```

В результате работы агента в папке нашего проекта появляется файл index.html, который мы двойным кликом открываем в браузере и любуемся первым результатом!

Примечание: получившуюся страницу можно легко дорабатывать - просто напишите агенту в чате свои пожелания и он подкорректирует где нужно.

Но это конечно же просто верстка. Чтобы оживить ее и сделать возможным сохранять данные с формы в базу данных, необходимо создать инфраструктурный слой нашего лендинга.

Для этого пишем агенту:
```
Лендинг index.html нужно преобразовать во frontend-приложение, которое будет локально запускаться в docker-контейнере.

Необходимо:
- создать локальный git репозиторий
- создать коммит текущего index.html лендинга
- описать docker-compose файл с контейнером app: node.js, express
- преобразовать лендинг в index.html во frontend приложение на typescript
- создать README.md с инструкцией запуска fronted-приложения с лендингом (запуск в detached режиме, остановка)
```

После того, как агент сделает свою работу, мы заходим в файл README.md и выполняем инструкции по запуску нашего лендинга в виде полноценного frontend-приложения, которое в дальнейшем будем обогащать различными функциями.
